# mesh_fusion 設計書 — RealityScan High Detail メッシュの GaussianWrapping 補完融合

日付: 2026-07-05 / 承認: ハシモトさん（対話にて設計承認済み）

## 目的

RealityScan (RS) の High Detail メッシュを主、GaussianWrapping (GW) のメッシュを補完材として、
**1ファイルの単一モデル**に融合する。RS は複数モデルの合成・一体テクスチャ生成ができないため、
ジオメトリ融合を本ツールが担い、**テクスチャは融合メッシュを RS にインポートして RS 側で生成**する。

## 決定事項（ヒアリング結果）

| 論点 | 決定 |
|---|---|
| 最終出力 | ジオメトリ融合のみ。テクスチャは RealityScan で生成 |
| 融合方式 | RS 優先 + GW 補完パッチ（RS ジオメトリは無劣化で保持、トポロジ非連結で可） |
| 検証データ | `D:\Claude\Photogrammetry\Sample_RS-ply\Project_____1.ply`（Sample_COLMAP と同一プロジェクト） |

## 前提

- GW メッシュは RS の COLMAP エクスポートから学習 → **原理上同一座標系**（無変換で重なるはず）
- RS のテクスチャ生成は投影方式 → 融合メッシュは連結多様体である必要なし、1ファイルであれば良い
- 入力メッシュは一切変更しない（非破壊）。半自動運用（各段階で人が確認）

## 処理フロー

### [1] 位置合わせ検証（診断のみが既定）
- GW→RS の最近傍距離統計（頂点サンプル N=100k）を算出しレポート
- 中央値 > 閾値（既定: RS 中央値エッジ長 × 20）なら**警告して停止**
- `--icp` 明示時のみ、スケール込み ICP（open3d）で **GW 側だけ**補正。RS は常に不動

### [2] GW 補完パッチ選定
1. **距離判定**: GW 各面の代表点（3頂点+重心）から RS 表面への距離 > **τ** の面を候補に。
   τ 既定 = RS 中央値エッジ長 × 8（CLI `--tau` で上書き可）
2. **ROI 制限**: RS メッシュ AABB を 10% 拡張した箱の外の面は除外（GW の背景破片対策）。
   `--roi_json` で GW Blender アドオンのバウンディングボリューム JSON も指定可
3. **最小成分フィルタ**: 候補面の連結成分ごとに面積を集計、
   合計面積 < RS 総面積 × 0.01%（`--min_patch_area_ratio`）の破片を除去
4. **オーバーラップ帯**: 採用パッチ境界から face adjacency を `--overlap_rings`（既定 3）回
   膨張して RS と少し重ねる（隙間防止。投影テクスチャでは重なりは無害）

### [3] 結合・出力
- RS メッシュ + 採用 GW パッチを単一メッシュに連結（頂点カラーがあれば保持）
- `fused.ply`（バイナリ）+ `fused.obj`（RS インポート互換の保険）
- `fusion_report.txt`: 距離統計 / τ / ROI / パッチ数・面積 / 除去破片数
- `preview_*.png`: RS部分=グレー、GW補完部=オレンジ着色で 2 視点レンダリング

## エラー処理

- 位置ズレ検出 → 既定は停止（明確なメッセージ + `--icp` の案内）
- パッチ 0 件 → 正常終了扱いだが「補完対象なし」と明記（RS が完全な場合の正しい挙動）
- 巨大メッシュ（RS 767MB 級）→ 距離計算は GW 面数側で駆動 + trimesh ProximityQuery
  （必要ならサンプリング分割でメモリ制御）

## テスト計画

1. **擬似データ自己検証**: GW メッシュから球状領域を人工削除した擬似RS を生成 →
   融合で削除領域が GW パッチとして復元されること、背景破片が混入しないことを定量確認
   （復元率 = 削除領域内の補完面積 / 削除面積、混入 = ROI 外採用面数 0）
2. **実データ検証**: Sample_RS-ply × GW メッシュ（`GW_Output\sample_ours`）で融合 →
   レポート・プレビューを人が確認 → RS インポート + Texture はハシモトさん側で最終確認
